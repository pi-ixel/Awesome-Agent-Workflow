## 执行规则
- 按串行执行顺序逐个执行（编号与执行顺序一致）
- 同一时刻只能开发一个任务，不允许并行
- 每个任务必须通过其 DoD checklist 才能开始下一个任务；唯一例外：当前 task 的核心验证已经通过，且剩余挂账用例不影响当前 task 主要完成状态时，挂账用例不阻断进入后续任务，待依赖任务完成后回归

## 串行执行顺序

对任务依赖图进行拓扑排序后确定的唯一执行序列，编号应与执行顺序一致。

示例：
```
T1 → T2 → T3
```
