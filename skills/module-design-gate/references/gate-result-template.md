# 模块详细设计门禁 context 模板

门禁以正式《{AR编号}-{需求短名}-{模块名}模块详细设计说明书.md》和独立《{AR编号}-{需求短名}-{模块名}模块测试用例设计.md》为准。正式说明书是后续 AICoding 拆分的设计依据，测试设计是后续验证实现的依据；`.context.md` 用于证据、过程复核和门禁记录。若开发或验证必需内容只存在于 `.context.md`，门禁必须判定不通过。门禁阶段不得编辑正式说明书或测试设计正文。门禁准入结论以维度化门禁检查为主，五链摘要和严格探索用于支撑、解释或反证维度结论。
